# Markdown 完整语法详解与实战指南

Markdown 是一种轻量级标记语言，允许人们使用易读易写的标准标记格式编写文档。在 Jekyll、Hexo 等静态网站生成器中，Markdown 是写文章的默认语言。

本文将详细梳理 Markdown 的完整语法规范，帮助你在博客写作中事半功倍。

## 1. 基础文本格式

基础文本用于控制标题、强调、删除线和文本颜色等。

### 标题 (Headers)
Markdown 通过 `#` 符号生成标题，级别从 1 到 6 级。
注意：6 级标题在实际渲染中往往会被隐藏或样式很小。

```markdown
# H1 - 一级标题
## H2 - 二级标题
### H3 - 三级标题

强调 (Emphasis)
使用星号 * 或下划线 _ 进行强调。

加粗：**文字** 或 __文字__
斜体：*文字* 或 _文字_
加粗且斜体：***文字***
删除线与文本颜色
Markdown 原生标准不支持删除线，通常依赖 CSS 插件（如 Kramdown 或 Rouge）支持。

删除线
文本颜色通常需要在 CSS 中定义（如 .highlight { color: red; }）
2. 代码与高亮
代码块的展示是技术博客的核心。

行内代码
使用单行反引号包裹，通常用于变量名。

这是一个 `print()` 函数的调用。
代码块 (Code Blocks)
使用三个反引号 ` 包裹。

无语言标记
console.log("Hello World");

带语言高亮 (推荐)
指定语言后，Jekyll 会调用 Rouge 或 Prism 进行语法高亮。

```javascript
function sum(a, b) {
    return a + b;
}
// 注释：这是一个简单的函数

### 行号与语法高亮 (HTML)
Jekyll 原生不支持行号显示，通常需要添加 HTML 元素或使用插件。
```html
<code class='ruby'>
def say_hi
  puts "Hi"
end
</code>
3. 列表与引用
有序列表 (Ordered Lists)
使用数字加句点 1.。

1. 第一步：安装依赖
2. 第二步：编写代码
3. 第三步：运行项目
无序列表 (Unordered Lists)
使用 -, * 或 +。

- 水果：苹果
- 水果：香蕉
- 水果：橙子
任务清单 (Task Lists)
大多数 Markdown 解析器支持勾选框（如 Kramdown）。

- [x] 已完成的任务
- [ ] 待完成的任务
引用 (Blockquotes)
使用 > 符号开头。

> 这是引用块的内容。
> 可以有多行引用。
>
> 引用支持加粗：**重要信息**
4. 链接与图片
链接 (Links)
[显示文本](目标 URL)

标准链接：GitHub
快捷链接（GitHub 短链接）：GitHub [blocked]
图片 (Images)
语法与链接几乎一样，但前面加上感叹号 !。

![alt text](路径/to/image.png)
链接形式：![](/assets/logo.png)
5. 表格与分割线
表格 (Tables)
需要三列以上，第一行是表头，最后一行是分隔符。

| 姓名 | 职业 | 爱好 |
| :--- | :--- | :---: |
| Alice | 程序员 | 编程 |
| Bob | 设计师 | 摄影 |
注意对齐方式：

:--- : 左对齐
---: : 右对齐
:---: : 居中对齐
分割线 (Horizontal Rules)
使用三个或更多的特殊符号：

---
***
***
___
渲染效果均为一条水平线。

6. HTML 嵌入与特殊字符
嵌入 HTML 标签
Jekyll 的 Markdown 解析器允许嵌入原始 HTML 标签，这极大地扩展了样式控制能力。

<!-- 红色文字 -->
<span style="color: red;">这是一个 HTML 红色文字</span>

<!-- 链接 -->
<a href="https://google.com">跳转 Google</a>
转义特殊字符
如果你想在文章正文中使用 # 或 * 等字符，需要转义（反斜杠 \）。

转义 #：#
转义 &：&
转义 <：<
转义 >：>
这是一个 # 标签 (转义为 \# 显示)
7. 提示框与任务清单 (Admonitions / Callouts)
部分 Markdown 处理器（如 Kramdown）支持自定义 HTML 标签或 CSS 样式。

在 CSS 中定义样式：

div.callout {
  padding: 1em;
  margin-bottom: 1em;
  border-radius: 5px;
  border-left: 5px solid #333;
}
div.callout.tip {
  border-left-color: #2196F3;
}
div.callout.danger {
  border-left-color: #f44336;
}
使用 HTML 标签（推荐）：

<div class="callout tip">
  <h4 style="margin: 0;">💡 提示</h4>
  <p>这是一个蓝色的提示框。</p>
</div>
结语
Markdown 的灵活性在于它可以在纯文本的基础上扩展出丰富的排版。建议在实际写作中多使用代码块高亮、表格和引用块，这些是技术博客的灵魂。

下一步建议：

探索 Jekyll 的 jekyll-toc 插件自动生成目录。
学习 Liquid 模板语言以在头部添加动态数据。
尝试使用 Hugo 或 Hexo 等其他工具。
